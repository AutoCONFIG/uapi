## 当前状态

Gateway / Relay 严格拆分已经完成到可验收状态：

- `uapi-gateway` 是 Web UI、admin/user API、API-key 鉴权、策略、计费、路由、Relay 管理控制面、Gateway proxy、debug dump 集中保存入口。
- `uapi-relay` 是执行节点，负责接收 Gateway 签名后的 `/v1/*`、`/v1beta/*` 执行请求，上游认证刷新、协议转换/透传、上游请求、stream 响应、usage/account/dump 回传。
- 不存在 all-in-one runtime mode、gateway/relay/all mode selector、`/internal/execute` execution wrapper、`OriginalURI` 路径恢复机制。
- Gateway -> Relay 数据面使用原始业务路径并做 HMAC 签名。
- Relay -> Gateway 控制面使用 `X-UAPI-Internal-Secret`。
- `/internal/*` 只作为控制面，不作为模型执行入口。

## 必须保持的架构边界

1. Gateway -> Relay 数据面必须保留原始业务路径：

```text
POST /v1/*
POST /v1beta/*
```

禁止恢复或引入：

```text
/internal/execute
internal/execute
OriginalURI
X-UAPI-Original-URI
HeaderOriginalURI
```

2. Relay 必须基于真实请求路径识别协议：

```text
/v1/responses
/v1/chat/completions
/v1/messages
/v1beta/*
```

不要依赖 internal wrapper 或 header 恢复路径。

3. 控制面路径只允许当前 split 设计：

Relay -> Gateway:

```text
GET  /internal/config
POST /internal/usage
POST /internal/account
POST /internal/dumps
```

Gateway -> Relay:

```text
POST /internal/reload
```

4. WebSocket 边界：

- 普通 `/v1/responses`、`/v1/chat/completions` 继续使用 HTTP/SSE。
- 不要把普通 Gateway -> Relay 请求转换为 WebSocket。
- WebSocket 只能作为 provider-native realtime/Codex 类专用能力。
- Relay 侧历史 WS 入口不能直接暴露公网用户。
- 如果要启用 WS，必须经过 Gateway 鉴权、策略、计费、路由和 HMAC 转发。

5. Debug dump 边界：

- Gateway 集中保存 dump。
- Relay production 推荐 `debug_dump.mode: "remote"` 或关闭 dump。
- Relay remote dump 通过 `POST /internal/dumps` 异步回传 Gateway。
- 不要减少现有 dump 类型。
- `index.jsonl` 应继续支持按 status/path/category/request_id/relay_node/channel/account/model 等维度过滤。

6. 协议转换边界：

- OpenAI Responses `input_image` 应转为 Anthropic `image` block。
- OpenAI Responses `input_file` PDF/document 应转为 Anthropic `document` block。
- Responses-only 字段不能泄漏到 Anthropic 请求，包括：

```text
input
input_image
input_file
image_url
file_type
mime_type
include
store
```

如遇上游协议疑点，先查本地 `upstream/` 中官方或竞品实现，不要凭空推测协议。

## Forbidden keyword 检查

正式代码和正式文档中不应出现旧机制推荐或实现。变更后用搜索工具检查并分类：

```text
server\.mode|all-in-one|runtime mode|mode selector|/internal/relay|internal/execute|/internal/execute|OriginalURI|X-UAPI-Original-URI|HeaderOriginalURI
```

可接受命中：

- `internal/relay` 作为 Go package/import/source tree path。
- `/internal/dumps` 和 `debugdump` 作为当前 remote debug dump/control-plane 设计。
- 本文件 `2.md` 或计划/审查提示文件中的 forbidden keyword 清单。

不可接受命中：

- 代码恢复 `/internal/execute`。
- HMAC 或 Relay handler 恢复 `OriginalURI`。
- 正式部署文档推荐 `/internal/relay/*` 或 all-in-one/runtime mode。
- 普通 `/v1/*` Gateway -> Relay 请求被改成 WebSocket 主链路。
